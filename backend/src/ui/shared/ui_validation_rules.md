***REMOVED******REMOVED*** UI Validation Rules (Operational Foundation)

These rules are used as a review checklist for Phase 1 operational UI work.

***REMOVED******REMOVED******REMOVED*** Visual overload

- No neon/glow AI visuals
- No marketing-style hero sections
- No excessive animations
- No excessive color usage; follow the calm token palette

***REMOVED******REMOVED******REMOVED*** Governance visibility

- Approval state must be visible (counts + link to Approval Center)
- Policy scope must not be hidden
- No “silent” or “auto” actions implied by the UI

***REMOVED******REMOVED******REMOVED*** Explainability visibility

- Evidence and lineage must have a clear surface (chip/badge/panel pointer)
- Timeline events must be readable chronologically with severity visibility

***REMOVED******REMOVED******REMOVED*** Operational states clarity

- Loading/empty/error states must be explicit
- Runtime status should be shown as a chip (ok/warn/critical)
- Risk indicators must be restrained and deterministic (no fake “AI confidence” theatrics)

***REMOVED******REMOVED******REMOVED*** Interaction scope (Phase 1 constraints)

- No advanced AI streaming UX yet
- No generative UI systems
- No autonomous action buttons
- Use deterministic metadata surfaces and existing endpoints
